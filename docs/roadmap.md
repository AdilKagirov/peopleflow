# KMF PeopleFlow Roadmap

## Phase 1: Foundation

- PostgreSQL schema.
- Seed reference data.
- Architecture and API contract.
- Static prototype kept as clickable UI reference.

## Phase 2: Backend Core

- Authentication.
- Roles and permissions.
- Vacancies API.
- Candidates API.
- Applications API.
- Pipeline movement history.
- Audit logs.

## Phase 3: Frontend Application

- Replace static localStorage prototype with API-backed UI.
- Vacancy list and editor.
- Candidate database and profile.
- Kanban pipeline.
- Interview scheduling.
- Reports.

## Phase 4: Files And Resume Parsing

- Resume upload.
- Attachment storage.
- PDF/DOC/DOCX text extraction.
- Parsed resume review screen.
- Candidate deduplication.

## Phase 5: Communications

- Email templates.
- Draft emails.
- Sending via corporate SMTP or mail API.
- Communication history.
- Interview reminders.

## Phase 6: Analytics

- Funnel conversion.
- Candidate sources.
- Time to close.
- Recruiter workload.
- Excel/PDF export.

## Phase 7: Legacy E-Staff Migration

- Analyze old E-Staff export.
- Build staging import.
- Map old IDs to new IDs.
- Validate counts and data quality.
- Import production data.
